Class: VcfRecord

This data class is used when it is desirable to pass data as expected from a VCF record. The class is only used as an optional attribute within a VariationDescriptor. The Genotype field from a VCF should be captured by the allelic_state attribute in the Variation Descriptor.

URI: GA4GHValueObjectDescriptorDefinitions:VcfRecord

Slots

Name Range Cardinality Description Info
genome_assembly NONE 1..1 Identifier for the genome assembly used to call the allele. .
chrom NONE 1..1 A chromosome or contig identifier. .
pos NONE 1..1 The reference residue-coordinate position, with the first residue having position 1. .
id NONE 0..1 A semicolon-separated list of unique identifiers where available. For example, dbSNP rsIDs. We RECOMMEND storing this information as a list in the VariationDescriptor xrefs field. .
ref NONE 1..1 Reference base as expected by the VCF specification. .
alt NONE 1..1 Alternate base as expected by the VCF specification. .
qual NONE 0..1 Quality: Phred-scaled quality score for the assertion made in ALT. .
filter NONE 0..1 Filter status: PASS if this position has passed all filters. .
info NONE 0..1 Additional information: Semicolon-separated series of additional information fields. .

Usages

used by used in type used
VariationDescriptor vcf_record range VcfRecord
